## 1. 시장 분석

### 1.1 국내 인플루언서 마케팅 플랫폼 시장 동향

- 비상장 스타트업·중소기업 중심이라 서비스 단위 매출·점유율 데이터는 거의 비공개이며, 트래픽·기사·후기 기반으로만 규모를 추정할 수 있다.
- 대표 플랫폼은 대략 세 축으로 나뉜다.
  - 체험단·리뷰형: 레뷰, 리뷰노트, 공팔리터, 링크튜브, 아싸뷰 등
  - SaaS형/B2B: 태그바이(TAGby), 피처링(Featuring), 미디언스, 해시업 등
  - 에이전시·커머스형: 더에스엠씨, 위블/페페로, 화해 크리에이터 마켓 등

- 체험단 시장은 리뷰노트가 월 실사용자 50만 명, 월 방문자 62만 명을 기록하며 "체험단 플랫폼 업계 1위"를 주장하는 등 빠르게 성장하고 있다.
- SaaS형 인플루언서 마케팅 도구로는 태그바이·피처링이 B2B 시장에서 선도적인 포지션을 차지하고 있으며, 태그바이는 셀프 서비스형 "New 태그바이"를 월 12만 원 구독형으로 출시한 바 있다.

## 3. 보완 방안

기존 플랫폼들이 놓치고 있는 부분을 신규 서비스가 보완하는 전략입니다.

### 3.1 시장 공백 분석 및 보완

#### 공백 1: "소상공인도, 대형 브랜드도 모두 쓸 수 있는 플랫폼의 부재"

**기존 문제:**
- 레뷰·리뷰노트: 체험단에 특화, 대형 브랜드의 정교한 타게팅·분석 기능 부족.
- 태그바이·피처링: 대형 브랜드 중심, 소상공인이 쓰기엔 UI·기능이 과하고 가격 정보가 불명확.

**신규 서비스의 보완:**
- **온보딩 시 광고주 유형 선택** (자영업 / 온라인몰 / 브랜드)에 따라 대시보드·옵션을 동적으로 조절.
- 소상공인 모드: 필수 항목만, 자동 매칭, 간단 리포트.
- 브랜드 모드: 고급 타게팅, 세분화 분석, API 제공 가능.
- **같은 DB·백엔드를 쓰되, UI/UX를 양쪽 모두에 최적화.**

#### 공약 2: "가격 정보의 불투명성"

**기존 문제:**
- 태그바이, 피처링, 미디언스 등 SaaS형은 거의 모두 "문의·견적형" → 초보 광고주 진입장벽 높음.
- 체험단형(레뷰 등)도 구체 상품별 가격 가이드는 많지 않음.

**신규 서비스의 보완:**
- **명확한 플랜 공시**: 스타터(월 5~10만) / 프로(월 20~30만) / 엔터프라이즈(커스텀) 구분.
- 각 플랜에 포함되는 캠페인 개수, 인플루언서 추천 수, 리포트 수준 명확 표기.
- **정부 바우처(중소기업혁신바우처 등) 호환성 설계** → 소상공인 실제 부담금을 50~95% 경감 가능.

#### 공약 3: "AI·데이터 활용의 불투명성"

**기존 문제:**
- "AI 추천", "데이터 기반" 같은 말은 많지만, 실제 알고리즘·정확도·한계를 설명하는 자료 부족.
- 광고주가 신뢰하고 사용하기 위해 필요한 '설명가능성(Explainability)'이 낮음.

**신규 서비스의 보완:**
- **인플루언서 추천 시 근거를 명시**: "카페 프로모션, 서울 강남역, 여성 20~35세, 인게이지먼트 3~5%" → "이 조건에 맞는 상위 10명".
- **품질 점수 공개**: "가짜팔로워 가능성 10%, 평균 인게이지먼트 4.2%, 동일 업종 이전 캠페인 5건".
- **성과 예측 범위 명시**: "예상 도달 30K~50K, 클릭 300~500, ROI ±20% 신뢰도 범위".

#### 공약 4: "인플루언서 경험의 제약"

**기존 문제:**
- 인플루언서는 여러 플랫폼을 병행 사용해야 하는데, 각 플랫폼마다 로그인·캠페인 확인·정산 조회가 분산됨.
- 기존 플랫폼들은 인플루언서 UX에 소상공인 광고주만큼 신경 쓰지 않음 (수익 구조상).

**신규 서비스의 보완:**
- **인플루언서용 "올인원 작업 대시보드"**: 캠페인 피드 → 지원 → 일정 관리 → 콘텐츠 제출 → 승인 → 정산까지 한 앱/웹에서.
- **자동 리마인더**: 마감 3일, 1일 전 알림. 미승인 콘텐츠 처리 알림.
- **투명한 정산**: 캠페인별 수수료, 세금 공제, 예상 입금액을 명확히 표시.
- **커뮤니티 기능** (추후): 인플루언서 간 정보 공유, 이전 캠페인 경험담 등으로 신뢰도 상승.

#### 공약 5: "로컬·오프라인 업체 맞춤 기능 부족"

**기존 문제:**
- 대부분 플랫폼이 "핸드폰 스크린에 보이는 콘텐츠(블로그·인스타)"에만 집중.
- 로컬 카페·식당·네일숍·헤어샵 같은 오프라인 점포는 실제 방문·구매 지표(예약 수, 내방객)를 추적하기 어려움.

**신규 서비스의 보완:**
- **지역 기반 필터링**: 캠페인 등록 시 "서울 강남역 500m 내 거주 인플루언서"를 타깃할 수 있게.
- **전환 추적 옵션**: 프로모션 코드 / QR 코드 / 예약 링크를 캠페인 물량에 자동 생성해서, 실제 오프라인 방문·구매를 카운트.
- **공동구매·어필리에이트 옵션**: 타다닥·공팔리터처럼, 단순 협찬을 넘어 실제 판매 수수료 구조 지원.

#### 공약 6: "대규모 데이터 부족으로 인한 AI 신뢰도 문제"

**기존 문제:**
- 초기 스타트업이 확보한 인플루언서·캠페인 데이터가 충분하지 않아 AI 모델의 신뢰도가 낮음.

**신규 서비스의 보완:**
- **MVP 출시부터 "데이터 우선" 설계**: 모든 지원·선정·콘텐츠 등의 메타데이터를 정구조화해서 저장.
- **초기 3~6개월은 AI 없이 규칙 기반 매칭** → 50건 이상 캠페인 데이터 축적 후 ML 모델 개발.
- **투명한 AI 로드맵 공시**: "현재는 규칙 기반, Q2부터 ML 적용, 성과 예측은 Q3부터" 같이 기대치 관리.

## 4. 차별점 및 강점

### 4.1 핵심 차별화 전략

#### 1️⃣ "소상공인 친화 + 대형 브랜드 확장성" 양축 설계

```
┌─────────────────────────────────────────────────────────┐
│                서비스 진입점 설계                         │
├─────────────────────────────────────────────────────────┤
│                                                           │
│  ┌──────────────────┐    ┌──────────────────┐           │
│  │  초보자 모드      │    │   전문가 모드     │           │
│  │ (소상공인 친화)   │    │ (대형 브랜드)     │           │
│  ├──────────────────┤    ├──────────────────┤           │
│  │• 5분 셋업       │    │• 다층 타겟팅     │           │
│  │• 필수항목만 입력 │    │• 고급 분석/리포트│           │
│  │• 자동 매칭       │    │• 브랜드 세이프티 │           │

│           ↓                        ↓                      │
│   ──────────────────────────────────                   │
│           공통 기반 (인플루언서 풀, 결제, 분석)           │
│   ──────────────────────────────────                   │
│                                                           │
└─────────────────────────────────────────────────────────┘
```

#### 2️⃣ AI 기반 자동 매칭 & 품질 관리

- **입력 5분 후 추천 인플루언서 세트 제시**
  - 업종·지역·채널·예산 기반 AI 학습
  - 기존 유사 캠페인 성과 데이터로 성공 확률 예측

- **자동 품질 필터링**
  - 가짜 팔로워 비율 자동 검출
  - 인게이지먼트 이상 패턴 탐지
  - 브랜드 가이드라인 위반 사항 사전 경고

#### 3️⃣ 비용 투명화 + 저진입장벽

    # 6. 상위 10명 반환

    # 키워드 추출
    keywords = extract_keywords(review_text)

    # 가이드라인 준수 체크
    contains_prohibited = check_prohibited_words(review_text)
    # 금지 단어, 과장 표현?
    
    # 종합 점수
    quality_score = (
        0.3 * (len(review_text) / 500) +  # 길이
        0.3 * (sentiment + 1) / 2 +        # 감정 (중립~긍정)
        0.2 * (len(keywords) / 5) +        # 키워드 다양성

    )
    
    return {
        "quality_score": quality_score,  # 0~1
        "sentiment": sentiment,
        "keywords": keywords,
        "issues": contains_prohibited,
        "recommendation": "APPROVE" if quality_score > 0.7 else "REVIEW"
    }

### 7.4 개발 구조 & 기술 스택 (MVP 기준)

#### 아키텍처 (Single Server 구조)

```
┌─────────────────────────────────────┐
│        Vercel / Netlify             │
│    (프론트엔드 호스팅, 자동 배포)   │
├─────────────────────────────────────┤
│                                     │
│  React/Vue.js + TypeScript          │
│  ├─ 광고주 대시보드                │
│  ├─ 인플루언서 대시보드             │
│  └─ 어드민 패널                    │
│                                     │
├─────────────────────────────────────┤
│        Backend API (Node.js/Express) │
│        (AWS EC2 or Render.com)      │
├─────────────────────────────────────┤
│                                     │
│  DB: PostgreSQL (AWS RDS)           │
│  Cache: Redis (캐싱, 세션)          │
│  Storage: AWS S3 (파일 저장)        │
│                                     │
├─────────────────────────────────────┤
│      Third-party Services           │
│  • 아임포트 (결제)                 │
│  • SendGrid (이메일)               │
│  • Slack (알림)                    │
│  • Twilio (SMS 옵션)               │
│                                     │
└─────────────────────────────────────┘
```

#### 기술 선택 (비용 최소, 신속성 최대)

| 계층 | 선택 | 이유 |
|-----|------|------|
| **프론트엔드** | React 18 + TypeScript + Tailwind CSS | 개발 속도, 커뮤니티, UI 라이브러리 풍부 |
| **백엔드** | Node.js (Express) + TypeScript | 빠른 개발, JavaScript 통일, 실시간 기능 용이 |
| **데이터베이스** | PostgreSQL | 복잡한 쿼리, 트랜잭션 처리, 확장성 |
| **캐시** | Redis | 세션, 카운터, 실시간 데이터 |
| **인증** | JWT + bcrypt | 상태 비저장, 보안 |
| **결제** | 아임포트 API | 한국 카드/계좌 지원, 보안 |
| **배포** | Docker + AWS EC2 or Render | 신속한 배포, 스케일링 가능 |
| **모니터링** | Sentry + ELK Stack | 에러 추적, 로그 분석 |

## 결론

### 시장 기회

국내 인플루언서 마케팅 시장은:
1. **소상공인 시장** (레뷰, 리뷰노트 등) - 높은 성장률, 낮은 기술 수준
